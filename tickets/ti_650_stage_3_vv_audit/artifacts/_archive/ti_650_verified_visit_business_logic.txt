<h1 id=3D"VerifiedVisitService(VVS)BusinessLogic-Glossary">Glossary&nbsp;</=
h1>
<p></p>
<p><em><strong>Verified Visit</strong> (<strong>VV</strong>) </em>: a singl=
e row in the <code>clickpass_log</code> that represents <span class=3D"inli=
ne-comment-marker" data-ref=3D"1b2f171e-d5ab-4d99-8f70-1e485bd5b1f6">a sing=
le non-click visit&nbsp;</span> to an advertisers website that was driven b=
y a MNTN ad that was served.&nbsp;</p>
<p></p>
<p><em><strong>Verified Visits Service</strong> (<strong>VVS</strong>)</em>=
 : is the microservice that is responsible for deciding whether a page view=
 can be attributed to a MNTN ad</p>
<p></p>
<p><em><strong>Verified Visits in MNTN reporting UI</strong> </em>: total c=
ount of visits in clickpass_log and click_log (where each row represents a =
single visit to an advertiser website by directly clicking on a MNTN MT ad =
that was served)</p>
<p></p>

n advertiser website with multiple page views.&nbsp;</em></p>

<h1 id=3D"VerifiedVisitService(VVS)BusinessLogic-Part1:Receivingaviewableim=
pression">Part 1: Receiving a viewable impression</h1>
<ol start=3D"1">
<li>
<p>Client wants to display an ad so they call the ad service with a serve r=
equest.</p></li>
<li>
<p>Ad service post this request for this impression in the kafka queue (imp=
ression topic).</p></li>
<li>
<p>Ad service also sends the Verified Visit Service (VVS) this request for =
an impression and logs it in users.click_pass (in scylla) where viewable =
=3D false.</p></li>
<li>
<p>Ad service serves the ad to the Client with an adcode</p></li>
<li>
<p>Client notifies event service that it has received a viewable ad with th=
at adcode&nbsp;</p></li>
<li>
<p>Once events receives that request, it calls VVS, which logs another impr=
ession in users.click_pass where viewable =3D true</p></li>
<li>
<p>Events service also logs that impression in the Kafka queue (impression =
topic)</p></li>

<h1 id=3D"VerifiedVisitService(VVS)BusinessLogic-Part2:TRPX,VVSandattributi=
on-consumerprocess">Part 2 : TRPX , VVS and attribution-consumer process&nb=
sp;</h1>
<ol start=3D"1">
<li>
<p>User sees an Ad on Tv, opens the browser ,on a device, and navigates to =
a webpage on the advertiser website that has a tracking pixel (TRPX) instal=
led.&nbsp;</p></li>
<li>
<p>This action triggers TRPX ,on the advertiser webpage, to fire. A record =
of this instance gets logged in guidv2.</p></li>
<li>
<p>TRPX sends an HTTP POST request to VVS every time it fires.&nbsp;</p></l=
i>
<li>
<p><strong>VVS</strong> responds to <strong>TRPX</strong> with true or fals=
e and logs true values in <code>clickpass_v2</code>. The response to <stron=
g>TRPX</strong> is <code>isSuccessful=3Dtrue</code> and contains impression=
 details when it's a <strong>Last Touch VV</strong> (<code>non-tamp</code> =
/ <code>attribution_model_id</code> is <strong>1, 2, 3</strong>). The respo=
nse is <code>isSuccessful=3Dfalse</code> when it's a <strong>First Touch VV=
</strong> (aka <code>tamp</code> VV / <code>attribution_model_id</code> is =
<strong>9, 10, 11</strong>)</p></li>
<li>
<p>TRPX sends ga_tracking_id, ga_client_id and relevant utm parameters of e=
ach verified visit to attribution-consumer.&nbsp;</p></li>
<li>
<p>Attribution-consumer sends this data to advertisers GA property via the =
measurement protocol api and logs a record of this in analytics_request_log=
.&nbsp;</p></li>
</ol>
<h1 id=3D"VerifiedVisitService(VVS)BusinessLogic-VVSDeterminationLogic">VVS=
 Determination Logic</h1>
<ol start=3D"1">
<li>
<p>VVS receives a POST request from TRPX each time it fires.&nbsp;</p></li>
<li>
<p>VVS verifies if the POST request contains a valid advertiser_id (aid) va=
lue.</p>
<ol start=3D"1">
<li>
<p>If aid value is not valid, VVS responds vv is false to TRPX (equivalent =
to <code>"isSuccessful":false</code> log in vv topic)</p></li>
</ol></li>
<li>
<p>VVS verifies whether the ip in the POST request is on a blocklist (segme=
ntation.ip_blocklist) : &nbsp;<em>1st stage of ip whitelisting.</em></p>
<ol start=3D"1">
<li>
<p>If ip is not on blocklist then VVS will proceed to cross device enabled =
check</p></li>
<li>
<p>If ip is on blocklist then VVS will proceed to the stage 1 of guid_white=
listing</p></li>
<li>
<p><em>segmentation.ip_blocklist is populated from Oracle-Audience-Service =
from the uploadBlockListById/{{uploadID}} endpoint. Users upload files of I=
Ps which are then uploaded to an S3 bucket. Table is stored in scylla.</em>=
</p></li>
</ol></li>
<li>
<p>VVS verifies whether the guid in the POST request is on a blocklist (seg=
mentation.guid_blocklist):<em>1st stage of guid whitelisting.</em></p>
<ol start=3D"1">
<li>
<p>If guid is on blocklist, VVS responds vv false to TRPX&nbsp;</p></li>
<li>
<p>If guid is not on blocklist VVS will proceed to the cross device enabled=
 check.&nbsp;</p></li>
</ol></li>
<li>
<p>VVS verifies whether an advertiser has crossdevice.config and advertiser=
s.clickpass_enabled enabled&nbsp;</p>
<ol start=3D"1">
<li>
<p>If advertisers.clickpass_enabled =3D false , vvs returns false to trpx</=
p></li>
<li>
<p>If the advertiser does not have crossdevice.config enabled, VVS only per=
forms same device attribution and not cross device attribution.</p></li>
<li>
<p>If the advertiser does have crossdevice.config enabled, VVS performs sam=
e device attribution and cross device attribution.</p></li>
<li>
<p><em>lookup table&nbsp; =3D vvs.cross_device_config in Aurora DB.</em></p=
></li>
</ol></li>
<li>
<p>VVS determines if the guid in the POST request matches a guid that was s=
erved an impression/ad within the VV window set by the advertiser (anywhere=
 between 14-45 days)&nbsp;(<em>by cross checking all the guid values in use=
rs.click_pass where viewable =3D true.)</em></p>
<ol start=3D"1">
<li>
<p>If match is found, VVS will proceed to check if guid is on blocklist (&n=
bsp;<em>2nd stage of guid whitelisting.)</em></p>
<ol start=3D"1">
<li>
<p>If guid is on blocklist, VVS will respond vv false to trpx&nbsp;</p></li=
>
<li>
<p>If guid is not on blocklist, VVS will proceed to eligibility check.&nbsp=
;</p>
<ol start=3D"1">
<li>
<p><em>If all other checks pass, this will be logged in clickpass_log as at=
tribution_model_id =3D 1 </em></p></li>
</ol></li>
</ol></li>
<li>
<p>If a match is not found, VVS will proceed to the cross device attributio=
n validation.</p></li>
</ol></li>
<li>
<p>VVS determines if the ip in the POST request matches a ip that was serve=
d an impression/ad in the clickpass window (<em>by cross checking all the i=
p values in users.click_pass where viewable =3D true)</em>.&nbsp;</p>
<ol start=3D"1">
<li>
<p>If match is found, VVS will proceed to check if ip was served a ctv impr=
ession. (<em>By checking if ip is on household_whitelist table in scylla</e=
m>&nbsp;)</p>
<ol start=3D"1">
<li>
<p>If ip was served a ctv impression, VVS will proceed to eligibility check=
.</p></li>
<li>
<p>If an ip was not served a ctv impression, VVS will proceed to the second=
 stage of ip whitelisting.&nbsp;(<em>filtering ips in icloud_ipv4)</em></p>
<ol start=3D"1">
<li>
<p>If ip is whitelisted, then VVS will proceed to eligibility check.&nbsp;<=
/p></li>
<li>
<p>If ip is not whitelisted, then VVS will lookup the number of guids mappe=
d to the ip.&nbsp;</p>
<ol start=3D"1">
<li>
<p>If count &gt;max then VVS responds false to TRPX&nbsp;</p></li>
<li>
<p>If count &lt; max then VVS will proceed to eligibility check.&nbsp;</p><=
/li>
</ol></li>
<li>
<p><em>If all other checks pass, this will be logged in clickpass_log as at=
tribution_model_id =3D 2</em></p></li>
</ol></li>
</ol></li>
<li>
<p>If a match is not found, VVS will proceed to ga_client_id attribution va=
lidation.</p></li>
</ol></li>
<li>
<p>Both guid matching and ip matching are repeated with impressions where v=
iewable =3D false</p></li>
<li>
<p>&nbsp;VVS determines if any of the ips associated with the ga_client_id =
(cookie.gaid_ip_mapping) in the POST request was served an impression/ad in=
 the clickpass window.(&nbsp;<em>by cross checking all the ga_client_id val=
ues in users.click_pass where viewable =3D true.)</em></p>
<ol start=3D"1">
<li>
<p>If a match is found, VVS proceed to eligibility check.&nbsp;</p>
<ol start=3D"1">
<li>
<p><em>If all other checks pass, this will be logged in clickpass_log as at=
tribution_model_id =3D 3</em></p></li>
</ol></li>
<li>
<p>If a match is not found, VVS will respond vv false to TRPX.&nbsp;</p></l=
i>
</ol></li>
<li>
<p>VVS checks if VV was logged for the same user earlier.&nbsp;(<em>Since T=
RPX sends POST requests every time it fires, it will send a request for eve=
ry page view event in a user Journey, typically only the first one will be =
eligible for verified visit consideration.)</em></p>
<ol start=3D"1">
<li>
<p>if&nbsp; it was, VVS responds vv false to TRPX.</p></li>
<li>
<p>If it wasn=E2=80=99t, VVS proceeds onto the next eligibility check.&nbsp=
;</p></li>
</ol></li>
<li>
<p>VVS determines if a user is eligible for the next VV.</p>
<ol start=3D"1">
<li>
<p>If they are, VVS responds vv false to TRPX.</p></li>
<li>
<p>If they are not, VVS proceeds onto the next eligibility check.</p></li>
<li>
<p>they are eligible if user (ip/guid/ga_client_id) been served an impressi=
on in the VV window stored in <code>clickpass_acquisition_ttl</code>that ha=
sn=E2=80=99t already been attributed to the same user in a previous site vi=
sit (even a tampered visit)</p></li>
</ol></li>
<li>
<p>VVS checks advertiser=E2=80=99s TTL configurations (<em>45 day max</em>)=
 from core-cache-service and determines if the request is within the window=
.&nbsp;</p>
<ol start=3D"1">
<li>
<p>If it is, VVS will proceed to a referral blocking check.</p></li>
<li>
<p>if not, then&nbsp; VVS will respond vv false to TRPX.&nbsp;</p></li>
</ol></li>
<li>
<p>VVS will check if the referrer in POST request of&nbsp;matched and eligi=
ble visit contains any substrings in the general list of tamp values (utm_s=
ource,utm_medium,utm_campaign,utm_content,gclid,cid,cmmmc)</p>
<ol start=3D"1">
<li>
<p>If substrings exist, then VVS responds vv false to TRPX</p></li>
<li>
<p>If substrings don=E2=80=99t exist, VVS responds vv true to TRPX.&nbsp;(e=
quivalent to <code>"isSuccessful":true</code> log in vv topic)</p></li>
</ol></li>
</ol>

<h2 id=3D"VerifiedVisitService(VVS)BusinessLogic-MisattributionandPV_GUID_L=
OCK"><a class=3D"external-link" href=3D"https://docs.google.com/document/d/=
1uQIKxuZfMdZdeLg9zdGaSjvvIdLvgyCgfUg0O8IaQXE/edit" rel=3D"nofollow">Misattr=
ibution</a> and PV_GUID_LOCK</h2>
<ul>
<li>
<p>VVS to store impression guid and pv guid&nbsp;</p>
<ul>
<li>
<p>To account for multiple ip changes in a site visit&nbsp;</p></li>
</ul></li>
<li>
<p>TTL for impression guid storage will remain unchanged while TTL for pv g=
uid storage will be set to 30 minutes</p>
<ul>
<li>
<p>To ensure pv guid remains eligible for verified visits outside of tamper=
ed site visit&nbsp;&nbsp;</p></li>
</ul></li>
<li>
<p>pv guid storage in expires after 30 minutes of <strong>inactivity</stron=
g> (TTL for page view guid will reset every time trpx calls with same pv gu=
id)</p>
<ul>
<li>
<p>To account for site visits longer than 30 minutes&nbsp;</p></li>
</ul></li>
<li>
<p>Advertisers with this logic applied will have feature flag pv_guid_lock =
=3D true in advertiser_configs table</p></li>
</ul>

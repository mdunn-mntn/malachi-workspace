# Marketing Channel Intermediate Attribution - SQL Challenge

## Problem Description

When a customer comes to Thumbtack searching for a particular service, they may end up creating a project, which is a detailed description of the work they would like to get done through our platform. Ultimately it is through these projects that we end up generating revenue.

Thumbtack works with several advertising partners to attract new customers and wants to know how the advertising dollars spent are translating into new customers generating projects, and ultimately revenue.

One of our advertising partners, the search engine "Findle" will directly provide ad click information for projects that we send them.

Our streaming ad partner, Adtube, sends us a list of ad impressions by IP address. We can get user IP addresses from their login information.

Use the information we have received from our multiple ad partners along with our own project and login data to show how much revenue was generated by each partner, and how much came to us unpaid.

## Requirements

1. Each row of the output table should provide a channel name and a total amount of revenue generated for the channel

2. If Findle has provided ad click information for a project, we will attribute the revenue for that project to the channel name "Findle"

3. If Adtube made an ad impression for a user within 7 days before creating a project, and there is no other channel attribution for this project, then the revenue for that project should be attributed to channel name "Streaming"

4. By default, projects are attributed the channel name "Unpaid"

5. Note that there can be multiple ad impressions or logins per user.

### Expected Results
```
channel    | total_revenue
-----------|---------------

Findle     | 900
Streaming  | 778
```

## Key Considerations

- **Attribution Priority**: Findle > Streaming > Unpaid
- **Time Window**: Streaming attribution only applies if impression was within 7 days BEFORE project creation
- **Exclusivity**: Streaming only gets attribution if there's NO Findle attribution for that project
- **Aggregation**: Sum all revenue by channel

## Hints

1. Consider using CTEs or subqueries to handle each attribution type separately
2. Be careful with the 7-day window calculation for Streaming
3. Use LEFT JOINs appropriately to check for absence of Findle attribution
4. Remember that a user may have multiple logins and multiple IP addresses
5. The final output should have exactly 3 rows (one for each channel type)

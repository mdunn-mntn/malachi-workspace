# Marketing Channel Advanced Attribution - SQL Challenge

## Problem Description

When a customer comes to Thumbtack searching for a particular service, they may end up creating a project, which is a detailed description of the work they would like to get done through our platform. Ultimately it is through these projects that we end up generating revenue.

Thumbtack works with several advertising partners to attract new customers and wants to know how the advertising dollars spent are translating into new customers generating projects, and ultimately revenue.

One of our advertising partners, the search engine "Findle" will directly provide ad click information for projects that we send them.

Our streaming ad partner, Adtube, sends us a list of ad impressions by IP address. We can get user IP addresses from their login information.

We receive ad impression information from our social media partner Imagehole. They index each ad click with an ID. If a user has come to our site via Imagehole, we associate this ID with that user's login.

Use the information we have received from our multiple ad partners along with our own project and login data to show how much revenue was generated by each partner, and how much came to us unpaid.

## Requirements

1. Each row of the output table should provide a channel name and a total amount of revenue generated for the channel

2. If Findle has provided ad click information for a project, we will attribute the revenue for that project to the channel name "Findle"

3. If a user has an Imagehole click id associated with their login information, and the associated ad click happened within 7 days prior to that user creating a project, attribute that revenue to the channel name "Imagehole"

4. **If a project can be attributed to ad clicks from multiple non-streaming partners, attribute it to the channel with the most recent associated ad click time**

5. If Adtube made an ad impression for a user within 7 days before creating a project, and there is no other channel attribution for this project, then the revenue for that project should be attributed to channel name "Streaming"

6. By default, projects are attributed the channel name "Unpaid"

7. Note that there can be multiple ad impressions, ad clicks, or logins per user

## Key Considerations

- **Attribution Priority**: 
  1. Most recent non-streaming partner (Findle or Imagehole)
  2. Streaming (only if no other attribution)
  3. Unpaid (default)

- **Time Windows**: 
  - Imagehole: ad click within 7 days BEFORE project creation
  - Streaming: impression within 7 days BEFORE project creation

- **Recency Rule**: When both Findle and Imagehole could claim a project, the one with the more recent ad_click_time wins

- **Login Properties**: The key 'imagehole_click_id' in login_properties links to imagehole_ad_clicks.in_click_id

## Hints

1. Use window functions (ROW_NUMBER() or RANK()) to handle the "most recent" attribution rule
2. Consider creating a unified view of all possible attributions with their timestamps
3. Be careful when joining imagehole data - you need to go through login_properties
4. The 7-day window applies to both Imagehole and Streaming
5. A single project should only be attributed to ONE channel
6. Use COALESCE or CASE statements to implement the priority logic
